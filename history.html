HISTORY OF FOSS

EARLIER THAN EARLY 1980s
- NORM OF PUBLIC DOMAIN SOFTWARE

In the 1950s - 1960s majority of the software was created by academics and business researchers working collaboratively. This software was often shared as public domain software and allowed users to access the source code (readable form of code that can be understood by humans with ease) alongside the machine code. As users had access to this code, they often modified the software themselves. This freedom enabled users to fix bugs, add new functions, change code so software can become compatible with different hardware and operating systems.

From the early 1970s onwards, software became copyrightable as the US Commission on New Technological Uses of Copyrighted Works (CONTU) made the decision that "computer programs, to the extent that they emobody an author's original creation, are proper subject matter of copyright". Collectively, court decisions alike Apple versus Franklin in 1983 for object code introduced the copyright status of code to computer programs, software now began becoming licensed and closed software started to become common.

Later in the 1970s and early 1980s, proprietary software such as IBM now sold their software at a price to increase revenue and decided to no longer distribute source code with the purchased software.

EARLY 1980s
- CHANGE BY STALLMAN

Proprietary software simply providing machine code compiled from the source code was becoming the norm. One man was particularly in despair by this change. That man was none other than Richard Stallman. He believed this act was ethically wrong as now his ability to further modify programs and to study code further was hindered.

As a result, Stallman introduced the concept of 'free software'. In 1982, he founded the GNU project enabling users to use their computer with only free software. Three years later, he formally established the GNU project by creating a non-profit organisation called the Free Software Foundation. Derived from copyright, Stallman innovated copyleft; a license to protect the 'free' status of one's work and allow users of their software to use their program as they wish to. Authors are advised from turning their free software into proprietary software for the values of remaining ethical.

POST EARLY 1980s
- THE INFLUENCE OF FOSS ON SOCIETY

After Stallman's stand for free software, many other people began creating free software alternatives to the proprietary software [read more here]. One would know if a software is 'free' if it complied with the four essential freedom principals as mentioned here.

In the late 1990s, the term "open source" was introduced by Eric S. Raymond. This is because open source was more of an umbrella term as it drifted awaty from the ethical and philosophical reasonings. Instead, it provides a term to reflect a more positive business value which promotes a collaborative and community-driven ethos. Thereafter, Raymond and others introduced Open Source Initiative[OSI] which extended the open source defintion. As stated here, there are ten principles a software license must practice in order to be classified as open-source.

Together, free software and open source combined to create Free[/Libre] and Open Source Software [FOSS/FLOSS]. The many people that still standby using FOSS are FOSS activists. These FOSS activists simply want to have the freedom to use software as they desire.
